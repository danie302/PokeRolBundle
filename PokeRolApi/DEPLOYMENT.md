# PokeRol API - Vercel Deployment Guide

This guide explains how to deploy the PokeRol API to Vercel.

## Prerequisites

- A Vercel account ([sign up for free](https://vercel.com/signup))
- MongoDB Atlas account (or another hosted MongoDB service)
- Email service credentials (SendGrid, Mailgun, or Mandrill)

## Quick Start

### 1. Prepare Your Environment Variables

Before deploying, make sure you have the following environment variables ready:

```bash
# Database (MongoDB Atlas recommended for production)
MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/pokerol?retryWrites=true&w=majority

# JWT Secret (generate a secure random string)
JWT_SECRET=your_secure_jwt_secret_here

# Email Configuration (example with SendGrid)
EMAIL_HOST=smtp.sendgrid.net
EMAIL_PORT=587
EMAIL_SECURE=true
EMAIL_USER=apikey
EMAIL_PASSWORD=your_sendgrid_api_key
EMAIL_FROM=noreply@pokerol.com

# Frontend URL (update with your actual frontend URL)
FRONTEND_URL=https://your-frontend-domain.com
```

### 2. Deploy to Vercel

#### Option A: Using Vercel CLI

1. Install Vercel CLI:
```bash
npm install -g vercel
```

2. Deploy to Vercel:
```bash
cd PokeRolApi
vercel
```

3. Follow the prompts:
   - Set up and deploy
   - Link to existing project
   - Configure your project name
   - Set up environment variables (use the values from step 1)

#### Option B: Using Vercel Dashboard

1. Go to [vercel.com](https://vercel.com) and log in
2. Click "Add New Project"
3. Import your repository (connect to GitHub, GitLab, or Bitbucket)
4. Configure project settings:
   - **Framework Preset**: Other
   - **Root Directory**: PokeRolApi
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`

5. Add environment variables from step 1
6. Click "Deploy"

### 3. Configure MongoDB Atlas

For production deployment, use MongoDB Atlas:

1. Sign up at [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. Create a new cluster
3. Create a database user with read/write permissions
4. Whitelist Vercel's IP ranges or allow all IPs (0.0.0.0/0)
5. Get your connection string
6. Update `MONGODB_URI` in Vercel environment variables

### 4. Configure Email Service

Choose an email provider and get your SMTP credentials:

**SendGrid** (Free tier available):
- Sign up at [sendgrid.com](https://sendgrid.com/)
- Create an API key
- Use SMTP settings with API key as password

**Mailgun** (Free tier available):
- Sign up at [mailgun.com](https://www.mailgun.com/)
- Create a domain and SMTP credentials

**Mandrill** (Requires Mailchimp):
- Sign up at [mailchimp.com](https://mailchimp.com/)
- Get Mandrill API key

## Environment Variables Reference

| Variable | Description | Example |
|----------|-------------|---------|
| `NODE_ENV` | Environment mode | `production` |
| `MONGODB_URI` | MongoDB connection string | `mongodb+srv://...` |
| `JWT_SECRET` | Secret for JWT token signing | `your_secret_here` |
| `EMAIL_HOST` | SMTP server hostname | `smtp.sendgrid.net` |
| `EMAIL_PORT` | SMTP port | `587` |
| `EMAIL_SECURE` | Use TLS | `true` |
| `EMAIL_USER` | SMTP username | `apikey` |
| `EMAIL_PASSWORD` | SMTP password/API key | `SG.xxxxx` |
| `EMAIL_FROM` | Sender email address | `noreply@pokerol.com` |
| `FRONTEND_URL` | Frontend URL for password reset links | `https://your-frontend.com` |

## Vercel Configuration

The project includes a `vercel.json` file with the following configuration:

```json
{
  "version": 2,
  "name": "pokerol-api",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "installCommand": "npm install",
  "framework": null,
  "functions": {
    "api/**/*.ts": {
      "runtime": "nodejs20.x",
      "maxDuration": 60
    }
  }
}
```

## Testing the Deployment

After deployment:

1. Get your Vercel deployment URL from the dashboard
2. Test the root endpoint:
```bash
curl https://your-project.vercel.app/
```

3. Test authentication endpoints:
```bash
# Login
curl -X POST https://your-project.vercel.app/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password"}'
```

## Troubleshooting

### Database Connection Issues

- Verify your MongoDB URI is correct
- Check MongoDB Atlas whitelist settings
- Ensure database user has proper permissions
- Review Vercel logs for connection errors

### Build Failures

- Check TypeScript compilation errors locally with `npm run build`
- Verify all dependencies are in `package.json`
- Check Vercel build logs for specific errors

### Runtime Errors

- Review Vercel function logs
- Check environment variables are properly set
- Verify database is accessible from Vercel's network
- Test API endpoints locally first

## Updating the Deployment

To update your deployed application:

1. Make changes to your code
2. Commit and push to your repository
3. Vercel will automatically detect changes and redeploy
4. Monitor the deployment in the Vercel dashboard

## Performance Optimization

- Enable Vercel Edge caching for static routes
- Use MongoDB Atlas for better performance
- Monitor function execution time (max 60s for serverless)
- Consider using Vercel KV for caching if needed

## Security Best Practices

- Never commit `.env` files
- Use strong JWT secrets
- Keep email credentials secure
- Enable MongoDB Atlas IP whitelisting
- Monitor Vercel logs for suspicious activity
- Set up rate limiting for API endpoints
